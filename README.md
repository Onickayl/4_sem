# Snake Game

## Описание проекта

Игра «Змейка» с поддержкой двух игроков, режимом ботов и графическим интерфейсом (SFML). Проект реализован на C++ с использованием паттерна MVC.


## Архитектура

Проект построен по паттерну MVC (Model-View-Controller):

- **Model** — хранит состояние игры (координаты змеек, кроликов, бонусов) и содержит всю игровую логику (движение, столкновения, поедание кроликов и бонусов)
- **View** — отвечает за отрисовку. Реализовано два варианта: `TerminalView` (ANSI-графика в терминале) и `SfmlView` (графическое окно через библиотеку SFML)
- **Controller** — связывает модель и представление, обрабатывает ввод с клавиатуры, управляет игровым циклом и таймером


## Алгоритмы

### 1. Алгоритм движения змейки

На каждом шаге змейка вычисляет новую позицию головы в зависимости от текущего направления. Затем проверяются столкновения: со стеной, с собственным телом и с другими змейками. Если столкновений нет, новая голова добавляется в начало списка сегментов. Хвост удаляется, если змейка не съела кролика или бонус (в этом случае счётчик роста уменьшается, а хвост остаётся). Направление сохраняется в поле `next_direction` и применяется только на следующем шаге — это исключает возможность мгновенного развороота на 180 градусов.

### 2. Алгоритм генерации бонусов

При съедании кролика увеличивается счётчик `eaten_rabbits`. На каждых 10 съеденных кроликов на поле появляется новое яблоко. На каждых 17 кроликов — новая снежинка. Через 3 кролика после появления яблока или снежинки (если их не съели) они удаляются автоматически.

### 3. Алгоритм работы тупого бота

Бот сканирует поле в поисках ближайшего объекта: сначала яблоки, затем кроликов. Расстояние вычисляется по манхэттенской метрике (`|x1-x2| + |y1-y2|`). Затем выбирается направление: сначала движение по горизонтали (влево/вправо), затем по вертикали (вверх/вниз). Единственное ограничение — запрет на разворот на 180 градусов. Бот не проверяет безопасность выбранного направления, поэтому может врезаться в стену или другую змейку.

### 4. Алгоритм работы осторожного бота

Сначала бот действует как тупой — выбирает направление к ближайшей еде. Затем проверяется до четырёх направлений в порядке приоритета: выбранное направление, поворот налево, поворот направо, исходное направление. Для каждого направления вызывается функция `is_safe()`, которая проверяет, не приведёт ли движение в этом направлении к столкновению со стеной, своим телом или другой змейкой. Первое безопасное направление применяется. Если все четыре направления опасны — бот не меняет направление.

### 5. Алгоритм изменения размера поля (TerminalView)

При каждом кадре контроллер получает текущий размер терминала через `ioctl()` и сравнивает с сохранённым. Если размер изменился, создаётся новая модель с новыми размерами. В новую модель переносятся все живые змейки (с сохранением их тела, направления и цвета), кролики и бонусы, которые оказались в пределах нового поля. Змейки, чья голова оказалась за границами поля, помечаются как мёртвые. Старая модель удаляется.


## Компиляция

g++ main.cpp model.cpp controller.cpp terminal_view.cpp sfml_view.cpp -lsfml-graphics -lsfml-window -lsfml-system -o snake


## Запуск

|Команда|	Режим|
|---|--|
|./snake manual	|Ручной режим, отображение в терминале|
|./snake manual sfml|	Ручной режим, графическое окно SFML|
|./snake test 10	|Тестовый режим, 10 прогонов ботов (терминал, без отрисовки)|
|./snake test 10 sfml|	Тестовый режим, 10 прогонов ботов (графика, с отрисовкой)|

## Режим ботов

|Тип бота|	Цвет	|Поведение|
|--|--|--|
|Тупой (bot_type = 1)|	Голубой|	Идёт кратчайшим путём к еде. Не проверяет стены и других змеек.|
|Осторожный (bot_type = 2)|	Жёлтый	|Проверяет безопасность направления перед движением. Выбирает первое безопасное направление из четырёх возможных.|
